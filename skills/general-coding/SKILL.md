---
name: general-coding
description: General coding guidance for writing or modifying code in any language. Use when implementing, refactoring, or reviewing code.
---

- Instead of comments:
  - make the code self documenting with descriptive variable and method names.
  - explain the reason for changes in commit messages
- Guard against null values only when necessary. When null values are likely, prefer appropriate type coercion.
- Commit as you are making changes (even if not prompted to). See git and commits skill when committing.
